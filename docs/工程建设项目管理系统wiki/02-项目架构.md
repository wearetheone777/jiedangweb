# 项目架构

## 整体架构

本项目使用 **uni-app** 框架开发，基于 Vue 2 运行时，采用 **单页应用(SPA) + 分包加载** 的架构模式。

```
┌─────────────────────────────────────────────────────────────────┐
│                       应用入口 (App.vue)                        │
│             全局样式 / 生命周期 / 公共资源加载                     │
├─────────────────────────────────────────────────────────────────┤
│                          main.js                                │
│     Vue 原型扩展 (canApprove / preview / downLoadFile 等)       │
│     Vue.mixin (setViewLogNavTitle)                              │
├────────────────────────────┬────────────────────────────────────┤
│      主包 pages             │        分包 homePack               │
│                            │                                    │
│  ┌──── home (首页) ────┐   │  进度管理   │  开工管理   │  安全管理  │
│  │  projectLog (项目列表)│   │  质量管理   │  投资管理   │  合同管理  │
│  │  itemInformation     │   │  问题管理   │  ...共40+页面 │         │
│  │  my (个人中心)        │   │                                    │
│  │  login (登录)         │   │                                    │
│  └──────────────────────┘   └────────────────────────────────────┘
├────────────────────────────┬────────────────────────────────────┤
│       工具层                │         UI 组件层                   │
│  utils/http.js (网络请求)   │   uni-ui 组件库 (官方扩展)          │
│  utils/baseurl.js (配置)    │   自定义组件 (项目专用)              │
│  api/* (业务 API 封装)      │   lime-echart (图表)               │
│  js_sdk/luch-request       │   静态资源 (images/fonts)           │
└────────────────────────────┴────────────────────────────────────┘
```

## 架构特点

### 1. 主包 + 分包策略

- **主包**：包含 4 个核心页面（首页、项目列表、项目详情、我的）和登录页，以及 Bottom TabBar（首页/项目列表/我的）
- **分包 `homePack`**：承载 40+ 业务功能页面，按业务模块拆分，实现按需加载

### 2. 基于 JWT 的身份认证

- 登录后服务端返回 `token`，存储在 `uni.getStorageSync('xmgl_token')`
- 每次请求自动在 `Authorization` header 附加 `JWT <token>`
- 401 响应自动清除登录态并跳转登录页

### 3. 审批流引擎

- 几乎所有业务单据都支持 **多级审批工作流**
- 审批状态流转：`un_submit`(未提交) → `approving`(审批中) → `approved`(通过) / `refused`(驳回)
- 通过 `approve()` 和 `approveLog()` 接口实现审批操作与审批记录查询
- `time-line` 组件统一展示审批时间线

### 4. 角色驱动的菜单

- 首页根据 `userInfo.dept_type` 动态渲染功能入口
- 通过 `verifyPermission()` 全局方法进行细粒度权限校验

### 5. 实时消息推送

- 使用 WebSocket 接收后端推送的审批消息、通知
- 利用 `plus.push.createMessage()` 在 APP 内创建本地推送消息
