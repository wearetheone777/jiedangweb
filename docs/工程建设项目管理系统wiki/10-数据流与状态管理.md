# 数据流与状态管理

## 数据流架构

本项目采用 **组件内状态 + 本地存储** 的状态管理模式，未引入 Vuex 或 Pinia。

```
┌───────────────────────────────────────────────────────────────┐
│                       后端 API (DRF)                          │
│                  http://192.144.191.81/api/                    │
└────────────────────────┬──────────────────────────────────────┘
                         │ JWT Token 认证
                         ▼
┌───────────────────────────────────────────────────────────────┐
│                   HTTP 层 (utils/http.js)                     │
│    luch-request + 拦截器（loading/token/401跳转）              │
└────────────────────────┬──────────────────────────────────────┘
                         │ JSON Response
                         ▼
┌───────────────────────────────────────────────────────────────┐
│                  API 封装层 (api/*.js)                        │
│                   业务逻辑接口封装                              │
└────────────────────────┬──────────────────────────────────────┘
                         │ async/await
                         ▼
┌───────────────────────────────────────────────────────────────┐
│                   Vue 组件 (pages/*.vue)                      │
│    data() + computed + methods + 生命周期                     │
│                                                               │
│    ┌─────────────────────────────────────────────┐            │
│    │             本地存储 (Storage)               │            │
│    │  xmgl_token  — JWT Token                   │            │
│    │  userInfo    — 用户信息(含角色/权限)         │            │
│    │  mobile-key  — 设备标识                     │            │
│    └─────────────────────────────────────────────┘            │
└───────────────────────────────────────────────────────────────┘
```

## 数据流方向

1. **用户操作** → 组件方法调用
2. **组件方法** → 调用 API 模块方法（如 `api.constructionLog.post(data)`）
3. **API 模块** → 通过 `http.request()` 发送请求
4. **请求拦截器** → 注入 Token、显示 Loading
5. **后端处理** → 返回响应
6. **响应拦截器** → 隐藏 Loading、处理 401/400 错误
7. **组件回调** → 更新本地 data、刷新页面

## 状态存储策略

### 本地存储 (uni.setStorageSync)

| Key | 存储位置 | 说明 |
|-----|---------|------|
| `xmgl_token` | main.js:8 / http.js:7 | JWT 认证令牌 |
| `userInfo` | main.js:8 / http.js:48 | 用户完整信息（含角色、权限列表） |
| `mobile-key` | http.js:10 | APP 设备标识 |

### 组件内状态

每个页面组件通过 `data()` 管理自身状态，包括：

- **列表数据**：`tableData` 数组
- **分页状态**：`search.page`、`search.page_size`、`next`（是否还有更多）
- **筛选条件**：`search.state`（审批状态）、`search.project`（项目ID）
- **UI 状态**：`navIndex`（Tab 索引）、`loading` 等

## 关键全局方法

以下方法在 `main.js` 中注册到 `Vue.prototype`，可全局调用：

### canApprove(data)
判断当前用户是否为指定单据的审批人：
```
返回 true 的条件：
  - 单据状态为 'approving'
  - 当前用户在 cur_approvers 列表中
```

### getUrlFileName(url)
从下载 URL 中解析文件名（`decodeURIComponent` 处理编码）。

### verifyPermission(name)
权限校验，检查 `userInfo.permissions` 是否包含指定权限标识：
```
使用方式：this.verifyPermission('project.delete,project.edit')
支持逗号分隔的"或"逻辑（满足任一即可）
```

### preview(url, urls)
调用 `uni.previewImage` 预览图片。

### downLoadFile(url)
文件下载流程：`uni.downloadFile → uni.saveFile → uni.openDocument`。

### setViewLogNavTitle(data, tit1, tit2) — mixin
在详情页中根据当前用户是否为审批人，动态设置导航栏标题。

## 组件间通信

| 方式 | 示例 |
|------|------|
| Props 向下传递 | `<file-update :media="true" @add="handleAdd">` |
| Events 向上通知 | `this.$emit('add', {id, url})` |
| 父组件直接调用子组件方法 | `this.$refs.chart.init(echarts)` |
| Storage 跨页面共享 | `uni.getStorageSync('userInfo')` |
| 路由参数 | `uni.navigateTo({ url: '...?id=123' })` / `onLoad(option)` |

## 审批流数据模型

```
单据状态流转:
  un_submit(未提交) ──→ approving(审批中) ──→ approved(已通过)
                         ↑                      ↓
                         └────── refused(已驳回) ──→ un_submit(重新提交)

审批记录模型:
  {
    approver: { id, nickname },   // 审批人
    result: true/false,           // 同意/驳回
    node_name: "省公司审批",       // 审批节点名称
    created_time: "2023-01-01",   // 审批时间
    remark: "同意"                 // 审批意见
  }
```
