# 运行方式

## 开发环境要求

- **HBuilderX**（DCloud 官方 IDE）：建议使用最新版本（4.x+）
- **Node.js**：12.x 以上
- **App 基座**：Android 或 iOS 真机 / 模拟器

## 运行步骤

### 1. 安装 HBuilderX

从 [DCloud 官网](https://www.dcloud.io/hbuilderx.html) 下载并安装 HBuilderX。

### 2. 导入项目

打开 HBuilderX，选择 `文件` → `导入` → `从本地目录导入`，选择项目根目录。

### 3. 配置后端地址

编辑 `utils/baseurl.js`，确认后端 API 地址配置正确：

```javascript
let baseURL = 'http://192.144.191.81/'
let uploadURL = "http://192.144.191.81/api/uploader/"
let wsURL = 'ws://192.144.191:8034/'
```

> **注意**：当前配置为开发/测试环境 IP 地址，发布前需替换为生产环境地址。配置位于 `utils/baseurl.js:3-5`。

### 4. 运行到 App

在 HBuilderX 中：
1. 连接 Android/iOS 真机或启动模拟器
2. 选择菜单 `运行` → `运行到手机或模拟器` → 选择目标设备
3. HBuilderX 会自动编译并安装 App 基座到设备

### 5. 登录验证

App 启动后会进入登录页，使用账号密码登录（调用 `api-token-auth/` 接口获取 JWT Token）。

## 打包发布

### Android 打包

在 HBuilderX 中：
1. 选择 `发行` → `原生App-云打包`
2. 配置 Android 签名证书
3. 勾选所需模块权限（manifest.json 中已预配）
4. 提交云打包服务

### iOS 打包

1. 选择 `发行` → `原生App-云打包`
2. 配置 iOS 证书（p12）和描述文件（mobileprovision）
3. 提交打包

## 注意事项

1. **跨域问题**：App 端不存在浏览器跨域限制，但 H5 调试时需注意
2. **权限配置**：Android 权限已在 `manifest.json` 的 `distribute.android.permissions` 中配置
3. **推送服务**：使用 uni-push 2.0，需在 DCloud 开发者中心配置
4. **包体积**：`lime-echart` 包含完整 echarts 库，注意对包体积的影响
5. **Vue 2 限制**：不支持 Vue 3 语法，代码中同时保留了 VUE2 和 VUE3 的条件编译分支
